  ---
  自动预测管线（共7步）

  ①数据采集

  datafc_provider 从多个数据源拉取：
  - Sofascore/datafc →实时数据（首选）
  - OpenLigaDB →赛程/结果（回退）
  - 缓存到 数据缓存/ 目录

  ②七维评分（L1基础层）

  对主客队分别打 7个维度 的分（每项1-10）：

  ┌────────────┬──────┐
  │    维度    │ 权重 │
  ├────────────┼──────┤
  │ 进攻能力   │ 20%  │
  ├────────────┼──────┤
  │ 防守稳固性 │ 20%  │
  ├────────────┼──────┤
  │ 中场控制力 │ 15%  │
  ├────────────┼──────┤
  │ 大赛经验   │ 15%  │
  ├────────────┼──────┤
  │ 战术适配度 │ 15%  │
  ├────────────┼──────┤
  │ 体能/状态  │ 10%  │
  ├────────────┼──────┤
  │ X因素      │  5%  │
  └────────────┴──────┘

  →加权算出 P_base（基础胜率）和 total_h/total_a（总分）

  ③xG + λ引擎

  - 从 xG 数据算 进攻强度/防守强度（attack_strength, defense_strength）
  - 用 adjust_lambda() 做攻防调整 →得出 λ_A、λ_B（预期进球）
  - 如果是淘汰赛 →λ×0.82（-18%，淘汰赛进球更少）
  - 如果有历史数据 →全局 λ校准（比对预期vs实际进球偏差）

  ④负二项分布模型（核心）

  用 NB(λ,θ)算出所有可能比分的概率：
  - 3-way：主胜/平局/客胜概率
  - 大2.5 / BTTS 概率
  - 前十比分分布（如 1-0: 12%, 1-1: 10%, 2-0: 8%...）

  ⑤M系数 + WVR 衰减（L2场景修正）

  根据比赛场景施加修正系数：
  - 淘汰赛 →×0.95
  - 德比 →×1.15
  - 轮换阵容 →×0.55~0.85

  WVR 衰减根据样本量打折：
  - A级（≥30场）→全额生效
  - B级（≥10场）→75%生效
  - C级（<10场）→50%生效

  →最终得到 M_coef →算出 P_final

  ⑥三级漏斗 + 偏差熔断（风控）

  强制闸门 G01-G14：赛程存在？数据源覆盖？方向合理？
  质量评分 Q01-Q18：扣分制，满分100
  偏差熔断：检测 BTTS+小2.25矛盾、冷门预警(#008)、低分检查(#026)

  →给出最终评级 ⭐⭐⭐ / ⭐⭐ / ⭐ / ❌ / ⛔

  ⑦输出 + 投注建议

  - 写入 预测数据/足球预测-2026-07-08-xxvsxx.md（⭐ 以上才存）
  - 写入 predictions_db.json
  - 自动拉赔率，用 betting_engine 算 +EV 和凯利比例
  - 输出完整市场分析：1X2、大小球、BTTS、角球

  OpenLigaDB 覆盖的赛事（uecl/uel/ucl/国家队杯赛） 只有赛程和比分，没有 xG 数据。要让自动预测引擎对这些赛事生效，需要：
  1. 从外部（如 FotMob、FBref）手动拉取球队的 xG/进球/失球数据
  2. 注入到系统的 team_strength_cache.json 和 elo_rankings_cache.json
  3. 在 TEAM_ELO_CODE 里添加队名→代码映射
  4. 然后跑 predict.py

  对于这类数据源薄弱的赛事，用 人工预测（quick_predict.py）会更省事——你直接填方向概率，不需要折腾数据注入。
