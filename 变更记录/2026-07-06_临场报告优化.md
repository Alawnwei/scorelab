# 临场报告优化 — 变更说明

> 修复 `--update` 模式的亚盘正负号误判问题和报告内容空洞问题。

---

## 修复1：亚盘对比逻辑（正负号归一化）

### 问题

初盘（odds-api）和临场（datafc/SofaScore）的 `ah_hdp` 正负号约定不同：

```
初盘: ah_hdp=0.75  → 主队让0.75（odds-api 主队视角）
临场: ah_hdp=-0.75 → 主队让0.75（datafc 可能用让球方视角）
```

旧代码直接比较数值：`0.75 - (-0.75) = 1.5` → 误报"强烈退盘"。

但实际上欧赔从 1.75→1.65（主胜赔率反而下降了），盘口根本没动。

### 修复

新增 `_ah_depth()` 归一化函数：

```python
def _ah_depth(hdp, odds_h, odds_a):
    """把亚盘归一化成【让球方视角】"""
    # 用赔率判断让球方（赔率更低的球队是让球方）
    favored = "home" if odds_h < odds_a else "away"
    depth = abs(hdp)          # 用绝对值比深度
    label = f"主让{depth}" 或 f"客让{depth}"
    return favored, depth, label
```

比较时：先比让球方是否一致 → 再比深度差。

### 效果

| 场景 | 旧行为 | 新行为 |
|:-----|:-------|:-------|
| +0.75 vs -0.75（符号不同，盘口实际未变） | 🔴 报"强烈退盘" | 🟢 "主让0.75 → 主让0.75，基本不变" |
| +0.50 vs +0.25（真正退浅） | 🟠 报"退浅" | 🟠 报"退浅0.25球" |
| 主让→客让（让球方逆转） | ❌ 无法准确判断 | 🔴 报"让球方逆转" |

## 修复2：报告内容升级

### 问题

旧报告只罗列赔率数字变化，不结合初盘方向给出解读：

```
主胜: 3.80→4.20 +10.5% 🔴 强烈
📋 建议: 主胜赔率+10.5%，建议关注
```

分析师看了不知道这个变化对自己的初盘判断意味着什么。

### 修复

1. **从 unified DB 中查找初盘的推荐方向**，输出时显示
2. **分析赔率变化对初盘方向的影响**：
   - 初盘推荐客胜 → 客胜赔率下降 → "利好，可维持"
   - 初盘推荐客胜 → 客胜赔率上升 → "需谨慎"
   - 初盘推荐客胜 → 客胜赔率不变 → "维持原判断"
3. **亚盘变化用自然语言描述**：如"主让0.75不变"、"退浅0.25球"、"让球方逆转"
4. **大小球变化指向性建议**：盘口上调→大球利好，盘口下调→小球利好

### 输出示例

```
  ========================================================
  临场信号: 🟢 稳定
  初盘方向: 客胜
  ========================================================
  主胜: 3.80 → 4.20  ⬆5.3%
  平局: 3.20 → 3.30  ⬆3.1%
  客胜: 1.90 → 1.85  ⬇2.6%
  亚盘: 主让0.50 → 主让0.25（退浅0.25球）

  📋 判断: 初盘方向「客胜」赔率下降2.6%，利好，可维持 | 亚盘主让不变
```

### MD报告同步

MD文件从"信号等级+"表格"改为："初盘方向+信号" + 赔率变化表 + 一段判断结论。
