## P1.1 淘汰赛 λ 修正

### 问题
淘汰赛阶段球队防守更保守、节奏更慢，场均进球比小组赛低 20-30%。  
当前 M 系数（淘汰赛 ×0.962 → WVR 后仅 -3.75%）力度完全不够。

### 修复
在 `predict.py` 的 λ 计算环节（`adjust_lambda` 之后）叠加淘汰赛进球抑制：

```python
KO_LAMBDA_FACTOR = 0.82  # λ × 0.82 = 预期进球 -18%
if dp.is_knockout_league(league):
    lam_h *= KO_LAMBDA_FACTOR
    lam_a *= KO_LAMBDA_FACTOR
```

### 影响（λ=2.00/1.50, θ=8 示例）

| 指标 | 小组赛 | 淘汰赛 | 变化 |
|:-----|:------:|:------:|:----:|
| λ_h | 2.00 | 1.64 | -18% |
| λ_a | 1.50 | 1.23 | -18% |
| 大2.5 | 65.1% | 53.0% | -12.1pp |
| BTTS | 62.2% | 52.8% | -9.4pp |

注：淘汰赛 λ 修正与 M 系数正交（λ 影响进球分布，M 影响胜率），不重复计算。

## P1.2 α + T 自动校准管线

### 问题
Logistic 斜率 α=0.15 硬编码在 `datafc_provider.py` 的 `P_base_raw` 计算公式中，从未被校准。

### 修复

1. **α 参数化**：`datafc_provider.py` 新增 `ALPHA_LOGISTIC = 0.15` 作为可配置变量
2. **DB 存储 score_diff**：`predict.py` 写入 predictions_db 时附带 `score_diff` 和 `M_coef`
3. **update_calibration.py**：
   - `read_current_alpha()` / `update_alpha()` — 读/写 α 到 datafc_provider.py
   - `find_best_alpha()` — 网格搜索 α∈[0.03, 1.00]，使用 score_diff 重算 P_base→P_final 路径
   - --apply/--auto 模式下 α 与 T 联合优化

当前 α 优化需 5+ 条含 score_diff 的记录（当前 0 条 → 静待新预测积累）。

### 文件变更

| 文件 | 操作 |
|:-----|:-----|
| `skill/datafc_provider.py` | +ALPHA_LOGISTIC 配置变量 |
| `skill/predict.py` | DB写入新增 score_diff, M_coef |
| `skill/update_calibration.py` | +α 搜索/读写/联合优化 |

DEFAULT_VIG = {

## P1.4 串关相关性折扣

### 问题
串关优化器 `optimize_parlay()` 将各场概率简单相乘（`parlay_prob *= p["model_prob"]`），假设场次完全独立。同联赛/同日的比赛存在正相关性，导致串关 EV 被高估 10-30%。

### 修复

1. **相关性折扣因子**：`_parlay_correlation_factor()` 按串关场数降低概率
   - 2 串：×0.90
   - 3 串：×0.81（0.90²）
   - 4 串：×0.73（0.90³）

2. **串关抽水随注数增加**：
   - 2 串抽水 ×1.12
   - 3 串 ×1.15
   - 4 串 ×1.18
   - 5+ 串 ×1.20

3. **串关仓位**：从 5% 降至 3%（更高风险 = 更低仓位）

4. **算法改进**：从"找最高赔率"改为"找最高+EV"，正确识别真实价值

### 影响
独立概率假设下高 EV 的串关可能被纠正为负 EV，减少虚假推荐。
