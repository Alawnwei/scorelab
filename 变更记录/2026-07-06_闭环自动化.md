# 闭环自动化 — 变更说明

> 每次 predict() 启动时自动回填 unified_predictions.json 中的 pending 赛果，
> 无需手动运行 sync_results.py。

---

## 修改

**文件：** `skill/predict.py`

### 新增函数
`_sync_pending_results()` — 在每次 `predict()` 执行前自动调用。

### 逻辑
1. 扫描 `unified_predictions.json` 中所有 `result.status="pending"` 的记录
2. 在 `auto_results.json` 中按队名模糊匹配
3. 若匹配到 `FINISHED` 状态→回填比分、方向正确性
4. 输出累计统计（总完赛场次、正确场次）
5. 若校准样本量达到50条→自动提示运行 calibration_v2.py

### 触发时机
`predict()` 函数第一条执行语句（打印标题后），在任何数据获取之前。

---

## 效果

```
以前：
  predict.py → 只做新预测
  sync_results.py → 需手动跑（经常忘记）→ 校准数据不增长

现在：
  predict.py → 先自动回填赛果 → 再生成新预测 → 校准数据自动增长
```

## 回滚方式
- 删除 `_sync_pending_results` 函数定义
- 删除 `predict()` 中的 `_sync_pending_results()` 调用
