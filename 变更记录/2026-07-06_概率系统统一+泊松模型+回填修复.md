# 概率系统统一 + 泊松模型 + 回填闭环修复

> **问题1：** 报告中出现两套概率（3-way Softmax 30.8/26.0/43.2% vs 泊松分布 40/21/40%），方向推荐和分布预测互相矛盾。
> **问题2：** `_sync_pending_results()` 只更新 unified_predictions.json，pnl_records.json 从未同步，PnL 永远 pending。
> **问题3：** λ 值直接用场均进球（λ_A=λ_B=2.0），不反映两队攻防差异。
>
> **方案：** 移除旧 Softmax 路径，统一为 attack/defense 分离的泊松模型 + 贝叶斯防守收缩。

---

## 改动文件

### 1. `skill/datafc_provider.py`

| 改动 | 说明 |
|:-----|:------|
| **`estimate_xg_from_openligadb()` 新增攻防强度因子** | 返回 `attack_strength`、`defense_strength`、`league_avg`，替代仅有 L1 评分的旧返回值 |
| **贝叶斯防守收缩** | `defense_shrunken = (GA×n + 1.0×3) / (n+3)`，防止 4 场零封→defense=0 的极端情况 |
| **修复 `NameError: attack_strength`** | 成功分支缺少赋值语句导致变量未定义 |

### 2. `skill/predict.py`

| 改动 | 说明 |
|:-----|:------|
| **移除旧 Softmax 3-way 路径** | 删除 `home_strength/away_strength → Logistic → softmax_3way` 的独立概率计算。现在 3-way 概率与泊松分布同源 |
| **新增 `poisson_3way()` 函数** | 从 λ 值计算泊松分布并推导主胜/平局/客胜概率 |
| **泊松 λ 新公式** | `λ_A = home_att_str × away_def_str × league_avg`，融合 ELO + H2H 修正 |
| **M 系数直接作用于 λ** | `λ_A *= M; λ_B /= M`（对称调整，替代旧 logit 空间调整） |
| **Step ② 存 attack_strength** | OpenLigaDB 路径将 strength 因子存入 `result["attack_strength_h/a"]` |
| **PnL 方向默认值修复** | P_final 0.45~0.55 不再写入"平局/均衡"，改为 `no_bet` |
| **`_sync_pending_results()` 同步 PnL** | 回填赛果后同时更新 `pnl_records.json`，计算盈亏 |
| **`_call_with_timeout` 超时 12→6s** | 加速 datafc 调用失败 |
| **所有 datafc 块加 DATAFIC_ALIVE** | 跳过不可达的 SofaScore |
| **Step ⑤ ELO 解绑 datafc** | `get_team_elo` 独立走本地缓存 |
| **Windows 编码修复** | `reconfigure(errors='replace')` 防止中文 print 崩溃 |

## 概率系统对比

### 改造前（两套打架）

```
              Softmax 3-way    泊松分布    市场赔率
主胜              30.8%         40%        23.5%
平局              26.0%         21%        26.0%
客胜              43.2%         40%        50.6%

→ 推荐方向: 主胜（来自 Softmax）
→ 最可能比分: 1-1（来自泊松）
→ 结论: 自相矛盾
```

### 改造后（统一泊松）

```
              泊松模型    市场赔率
主胜             7.0%     23.5%
平局            24.2%     26.0%
客胜            68.8%     49.5%

→ 推荐方向: 客胜 ✅（三项基线一致）
→ 最可能比分: 0-1 ✅
→ λ_A=0.30 λ_B=1.54（攻防分离）
→ 结论: 统一，无矛盾
```

## PnL 闭环修复

### 改造前
```
predict() → unified_predictions.json (result.status=finished) 
         → pnl_records.json (result=pending 永远不更新)
```
PnL 记录 45 条，0 条已结算，盈亏永远为 0。

### 改造后
```
_sync_pending_results() → unified_predictions.json (更新赛果)
                        → pnl_records.json (同步: win/loss + pnl)
```
PnL 记录 23 条(去重后)，10 条已结算（1W/9L, -2.85u）。
