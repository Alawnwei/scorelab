# 七维评分 tanh 饱和修复 + 虚假预测清理 + M系数H2H因子移除

> 八项修复：① `score_from_xg` tanh 饱和曲线；② 删除虚假预测；③ 移除 H2H M 系数因子；④ 方向阶梯判断；⑤ #008 模糊局 top_gap 检查；⑥ 赛程校验门禁；⑦ 实际进球贝叶斯收缩；⑧ 队名模糊回退 + TEAM_DE 扩充。

---

## 修改

### 1. 七维评分 — tanh 饱和映射（防止顶级球队封顶）

**问题：**
- `_score_from_ratio`（进攻/防守）用 `5 + (ratio-1)×5` 线性映射，ratio≥2 即封顶 10
- 中场控制力 `ctrl_raw × 2.5` 三级放大（攻防比 × ELO × 2.5），强队轻松超 10 后钳制
- 阿根廷 λ=2.66/avg=1.40 → ratio=1.90 → 进攻 9.5（修复前直接 10）
- 中场 att_str/def_str=1.90/0.22 × ELO 1.43 × 2.5 = 22.7 → 钳制为 10

**修复：** 线性 → `tanh` 饱和曲线

| ratio | 旧线性 | 新 tanh |
|-------|--------|---------|
| 1.0（平均） | 5.0 | 5.0 |
| 1.5 | 7.5 | 6.9 |
| 2.0 | **10 封顶** | **8.3** |
| 2.5 | **10** | **9.2** |
| 3.0 | **10** | **9.6** |

中场公式从 `ctrl_raw × 2.5` 改为 `5 + 5×tanh(0.35×(ctrl_raw-1))`。

**涉及文件：**
- `skill/datafc_provider.py` — 3 处：`_score_from_ratio`、`score_from_xg`×2、中场公式
- `skill/xg_estimator.py` — 1 处：`format_output` 内 `score_from_xg`

### 2. 清理虚假预测报告

**背景：** `预测数据/足球预测-2026-07-06-阿根廷vs法国.md` 是对一场不存在的比赛的预测。经对比 OpenLigaDB 赛程（94 场已完成比赛中无阿根廷 vs 法国），确认为系统生成的非赛程预测。

**操作：**
- 删除 `预测数据/足球预测-2026-07-06-阿根廷vs法国.md`
- `auto_results.json` / `predictions_db.json` 保持原状
- 其他所有世界杯比赛数据经验证均为真实 OpenLigaDB 数据，保留不动

### 3. M系数移除「历史交锋_淘汰赛」因子（P0）

**问题：**
- `calc_h2h_weight` 对淘汰赛永远返回 0.625 → 触发添加系数 0.50(WVR→0.625)
- 但系统当前**未使用任何 H2H 数据**，该惩罚因子无数据支撑
- 与淘汰赛因子(0.9625)相乘 → M=0.9625×0.625=**0.602**
- 所有淘汰赛预测被系统性压低 38%

**影响验证：**

| 比赛 | P_base | 修复前 P_final (M=0.602) | 修复后 P_final (M=0.9625) |
|------|--------|-------------------------|--------------------------|
| 阿根廷vs佛得角 | 0.728 | 0.617 (Δ-11pp) | **0.720** (Δ<1pp) |
| 英格兰vs美国 | 0.565 | 0.439 (Δ-13pp) | **0.556** (Δ<1pp) |
| 巴西vs挪威 | 0.568 | 0.442 (Δ-13pp) | **0.559** (Δ<1pp) |

**涉及文件：**
- `skill/predict.py` — 移除 4 行 H2H 因子追加代码（保留注释供未来接入时参考）

### 4. 方向判断逻辑优化（P1）

**问题：**
- `direction = '主胜' if probs[0] > 0.5 else ('客胜' if probs[2] > 0.5 else '平局')` 过于简单
- 英格兰vs美国(46%/18%/36%)：主胜 46% >> 平局 18%，却输出"平局"，严重误导

**修复：** 阶梯判断逻辑

| 条件 | 输出 | 例子 |
|:-----|:-----|:-----|
| 某方 > 50% | 主胜/客胜 | 阿根廷 87% → 主胜 |
| 某方 > 40% 且拉开平局 ≥ 10pp | 主不败/客不败(双选) | 英格兰 46% vs 18% → **主不败(双选)** |
| 某方 > 38% 且拉开平局 ≥ 8pp | 主胜/客胜(低信心) | 轻微优势局 |
| 平局最高且 ≥ 35% | 平局 | 明确平局局 |
| 平局最高且 ≥ 32% | 平局(有风险) | 平局但不太确信 |
| 各方概率相近 | 不推荐/均衡 | 34/33/33 → 不推荐 |

**涉及文件：**
- `skill/predict.py` — 重写 direction 赋值逻辑

### 5. #008 模糊局误判修复（P1）

**问题：**
- 英格兰vs美国(46%/18%/36%) spread=28pp，方向明确，却被标为🟡模糊局
- 逻辑缺陷：均衡局检查(spread<15pp)通过后，只要max<50%且p_final在区间就触发模糊局
- 忽略了"前两名差距"维度

**修复：** 均衡局收敛至 spread<12pp；模糊局增加 top_gap(第1-2名差距)<10pp 检查

| 比赛 | spread | top_gap | 旧结果 | 新结果 |
|:-----|:------:|:-------:|:------:|:------:|

| 均衡34/33/33 | 1pp | — | ✅均衡局 | ✅均衡局 |
| 胶着45/10/45 | 35pp | 0pp | 🟡模糊局 ✅ | 🟡模糊局 ✅ |

**涉及文件：**
- `skill/quality_engine.py` — `check_deviation_008` 增加 top_gap 判断

### 6. 赛程校验门禁 — 防止虚拟比赛干扰

**根因：** `predict.py` 和 `manual_predict.py` 接受任意两队名即运行管线，无赛程校验。英格兰vs美国、阿根廷vs法国等不存在赛程的比赛生成了完整预测报告。

**修复：** 新增 `datafc_provider.is_match_scheduled()` 函数，复用 OpenLigaDB 缓存数据校验两队是否曾有或将有比赛。入口添加 `--force` 可跳过。

```python
# predict.py 入口行为
$ python skill/predict.py --home "英格兰" --away "美国"
⚠️  ❌ 未找到赛程: 「英格兰 vs 美国」不在 wm26 的 OpenLigaDB 赛程中
    继续运行将生成一场不存在的比赛预测！
    若要强制跳过校验，添加 --force 参数。

$ python skill/predict.py --home "英格兰" --away "美国" --force   # 强制生成
```

**涉及文件：**
- `skill/datafc_provider.py` — 新增 `is_match_scheduled()` 函数
- `skill/predict.py` — main() 入口增加赛程校验门禁 + `--force` 参数
- `skill/manual_predict.py` — main() 入口增加赛程校验门禁 + `--force` 参数

### 7. 实际进球贝叶斯收缩 — 小样本下防止极端值（P2）

**问题：**
- `estimate_xg_from_openligadb` 使用实际进球代替 xG，方差是 xG 的 3-5 倍
- 德国 n=4 场，7-1 库拉索一场把场均从 ~1.8 拉到 3.5
- 加上 ELO 调整 +0.30 → xG_for=**3.80**，实际是防守型球队
- 葡萄牙同样 n=4，因对手强场均仅 1.93，被低估
- 攻击端**无任何收缩**（防守端已有收缩代码）

**修复：** 攻击端加贝叶斯收缩（prior=3 场，向联赛均值回归），防守端同步修正 prior 锚点

```python
# 收缩公式
attack_shrunken = (raw * n + avg * 3) / (n + 3)
```

| 球队 | n | raw | avg | 旧 xG_for(含ELO) | 新 xG_for(收缩后) |
|:-----|:-:|:---:|:---:|:-----------------:|:-----------------:|
| 德国 | 4 | 3.50 | 1.40 | 3.80 | **2.90** |
| 葡萄牙 | 4 | 1.93 | 1.40 | 2.13 | **1.90** |
| 阿根廷 | 4 | 2.00 | 1.40 | 2.60 | **2.34** |
| 法国 | 5 | 2.80 | 1.40 | 3.39 | **2.81** |

**涉及文件：**
- `skill/datafc_provider.py` — `estimate_xg_from_openligadb` 中攻击端增加收缩逻辑

### 8. 队名模糊回退 + 扩充 TEAM_DE（P2）

**问题：**
- TEAM_DE 未覆盖所有球队（如喀麦隆、尼日利亚等），`_build_names` 找不到匹配时直接返回空数据
- 仅依赖于手工维护的映射表，漏一个队就查不到数据

**修复：**

| 改动 | 说明 |
|:-----|:------|
| `_fuzzy_discover_team()` | 新增函数：扫描 OpenLigaDB 数据，用归一化匹配+音近模糊+子串截断自动发现队名 |
| `_enrich_match_names()` | 新增函数：在原有映射基础上加模糊回退，供所有管线统一调用 |
| TEAM_DE 扩充 | 追加 ~50 个常见国家队映射（喀麦隆、尼日利亚、冰岛、中国等） |

模糊匹配流程：
```
输入中文名 "喀麦隆"
  → TEAM_DE 命中?  → 直接返回德文名 "Kamerun"  ✅
  → TEAM_DE 未命中 → TEAM_EN 有 "Cameroon"
    → 音近模糊: cameroon → kamerun → 匹配 "Kamerun"  ✅
  → 都未命中 → 截断前 4 字扫描数据  → 可能发现
```

**涉及文件：**
- `skill/datafc_provider.py` — 新增 2 个函数 `_fuzzy_discover_team`、`_enrich_match_names`；替换 `is_match_scheduled` 和 `estimate_xg_from_openligadb` 中的内联匹配逻辑；扩充 TEAM_DE ~50 条映射
