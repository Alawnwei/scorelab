# 基线对比 + 复盘方式切换 — 变更说明

> 第二次诊断后实施的两项改造。修改1为零风险纯加法，修改2为过程改进。

---

## 修改1：基线对比制度

**文件修改：** `skill/predict.py`

### 新增函数
- `compute_baselines(home, away, result)` — 计算ELO排名基线和市场赔率基线

### 修改内容

| 位置 | 改动 |
|:-----|:-----|
| 第906-907行 | ELO数据存入 `result["elo_home"]` / `result["elo_away"]` |
| 第1251-1256行 | 预测流程中调用 `compute_baselines()` 并打印到控制台 |
| 第1312-1313行 | 报告模板增加基线对比行 |

### 输出示例

**控制台：**
```
基线: ELO=[法国] 赔率=[法国] 模型=[主胜]
一致性: ✅ 三项一致
```

**报告中：**
```
| 基线对比 | ELO=[法国] 市场=[法国] 模型=[主胜] |
| 一致性 | ✅ 三项一致 |
```

### 一致性等级

| 等级 | 含义 |
|:-----|:------|
| ✅ 三项一致 | ELO+市场+模型都指向同一队（高确信信号） |
| ⚠️ 模型与一个基线一致 | 模型与ELO或市场之一一致 |
| 🔴 模型与双基线均分歧 | 模型方向与ELO和市场都相反（潜在预警） |
| — 平局推荐 | 平局方向不参与基线对比（基线无法推荐平局） |

**验证：** 通过了5个场景测试（三项一致、双基线分歧、平局、无ELO、全指向客队）

## 修改2：复盘方式切换

**文件修改：** `skill/checklist_manager.py`
**文件修改：** `数据缓存/checklist.json`

### 改动本质

从 **"每次发现错误→立即加一条新规则"** 切换到：
**"每次发现错误→记入待观察清单→积累50场比赛后批量验证→通过才升级为规则"**

### 新增/修改函数

| 函数 | 说明 |
|:-----|:------|
| `add_observation()` | **替代旧的 `add_finding()`**。将新偏差记入待观察，不加入规则列表 |
| `list_observations()` | 列出所有待观察项 |
| `print_observations()` | 打印待观察清单 |
| `validate_observations()` | 批量验证待观察项，通过的升级为规则 |

### checklist.json 新增结构

```json
{
  "observations": [...],        // 待观察项（替代直接加到 items）
  "observation_meta": {
    "batch_size_target": 50,    // 积累目标
    "current_batch_count": 2   // 当前数量
  }
}
```

### 使用流程

```
赛后偏差分析
  ↓
发现一个新偏差模式
  ↓
调用 add_observation()                     ← 不再调用 add_finding()
  ↓                                   
积累到50场比赛后
  ↓
调用 validate_observations()               ← 批量验证
  ↓                ↓
验证通过                  验证未通过
  ↓                      ↓
升级为规则                  废弃
  ↓                      ↓
进入checklist.json.items    无变化
```

### 回滚方式

- `add_observation` → 恢复旧的 `add_finding`（如果有需求）
- 如果要完全回到旧模式，删除 `observations` 和 `observation_meta` 字段即可
