# 关闭校准 + 数据质量门槛 — 变更说明

> 修复两项当前版本的积弊：校准参数与模型脱节、低质量数据仍输出星级。

---

## 修复1：暂时关闭Platt校准

**位置：** `skill/predict.py:280-293`

### 背景

权重从40/40/20改为50/50后，score_diff分布发生变化。旧校准参数（CALIB_A=0.60, CALIB_B=-0.95）是在旧权重下学习的，继续使用会引入系统性偏差——可能把正确的概率压得过低或拉得过高。

### 修改

```diff
- CALIB_A = 0.60   # 斜率（<1 = 降低置信度）
- CALIB_B = -0.95  # 截距（负值 = 整体下调）
+ CALIB_A = 1.0    # 临时关闭校准（1.0=不缩放）
+ CALIB_B = 0.0    # 临时关闭校准（0.0=不偏移）
```

### 效果

`platt_calibrate(p)` 现在返回 `≈p`（浮点误差范围内不变），相当于跳过校准步骤。

### 恢复条件

积累50场以上有效的预测+赛果数据后，运行 `calibration_v2.py`，将输出的新参数写回：

```python
CALIB_A = <新斜率>
CALIB_B = <新截距>
```

## 修复2：数据质量门槛

**位置：** `skill/predict.py:1128-1146`

### 背景

quality_score 被计算出来并打印警告，但从未影响最终输出。即使是 quality=45 的预测（基于联赛默认xG、无赔率、小样本），照样输出 ⭐⭐ 评级，给分析师虚假的信任感。

### 修改

在星级评定前增加门槛检查：

```python
if quality < 60:
    result["star_rating"] = "⚠️ 数据不足"
    # 跳过星级计算
else:
    # 正常计算星级
```

### 效果

| quality_score | 之前 | 之后 |
|:-------------|:-----|:-----|
| 85+ | ⭐⭐⭐ | ⭐⭐⭐ |
| 70-84 | ⭐⭐ | ⭐⭐ |
| 60-69 | ⭐ | ⭐ |
| **< 60** | **仍输出⭐或⭐⭐（虚假）** | **⚠️ 数据不足（诚实标记）** |

---

## 回滚方式

1. **校准：** 将 CALIB_A/CALIB_B 改回旧值或校准脚本给出的新值
2. **质量门槛：** 删除 `if quality < 60` 条件分支，恢复旧星级计算
