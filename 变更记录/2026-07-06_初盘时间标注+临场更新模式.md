## 修改3：`--update` 临场更新模式

**新增参数：** `predict.py` 的 `--update` 参数
**新增函数：** `run_update()` — 临场更新主流程
**新增写入：** `unified_predictions.json` 中 `source="prematch_update"` 的新记录

### 用法

```bash
# 先跑初盘
python predict.py --home "墨西哥" --away "英格兰"

# 赛前任意时间跑临场更新
python predict.py --home "墨西哥" --away "英格兰" --update
```

### 流程

```
① 读取初盘赔率缓存（odds_墨西哥_英格兰_日期.json）
② 抓取当前实时赔率（datafc → odds-api 回退）
③ 对比初盘 vs 当前 → 四级信号判定
④ 输出修正报告（控制台 + MD）
⑤ 写入 unified_predictions.json 作为新记录
```

### 四级信号系统

| 信号 | 触发条件 |
|:----:|:---------|
| 🟢 稳定 | 所有赔率变化 < 5% |
| 🟡 关注 | 任一赔率变化 5-10% |
| 🟠 警惕 | 变化 10-15% 或亚盘退/升一档 |
| 🔴 强烈 | 变化 >15% 或盘口跨档 |

### 设计原则

- **只报警不自动改方向** — 修正建议是"建议留意/建议下调信心"，不做自动翻转
- **不修改初盘记录** — 在 unified 中追加新记录（`source="prematch_update"`），保留完整决策链
- **同一小时内重复跑覆盖更新** — 避免多轮修正产生大量冗余记录
- **初盘缓存不存在时报错退出** — 防止无基准的误更新

### 数据格式

unified_predictions.json 中的临场更新记录：

```json
{
  "source": "prematch_update",
  "match": {"home": "墨西哥", "away": "英格兰"},
  "odds": {"home": 4.20, "away": 1.85},
  "update": {
    "signal": "警惕",
    "changes": {"ah": {"from": 0.5, "to": 0.25, "direction": "退盘", "signal": "警惕"}},
    "initial_time": "2026-07-05 08:18",
    "initial_id": "a1b2c3d4"
  },
  "result": {"status": "pending"}
}
```

复盘时通过 `source` 区分初盘和临场，校准可分别统计。

### 边界情况

| 场景 | 处理 |
|:-----|:------|
| 无初盘缓存 | 报错退出，提示先不带 --update 跑预测 |
| 临场API不可达 | 报错退出，建议手动确认 |
| 同一比赛跑多次 --update | 同一小时内覆盖，不同小时追加 |
| 临场赔率来源与初盘不同 | 只对比两者都有的字段 |

### 风险

| 风险点 | 等级 | 说明 |
|:-------|:----:|:------|
| 临场赔率来源与初盘不一致 | 🟡 中 | 已做字段存在检查，缺失字段跳过 |
| 信号误报（盘口波动≠方向反转） | 🟢 低 | 只报警不自动改，人做最终决定 |
| 影响现有predict()流程 | 🟢 无 | 纯新增分支，原有路径不变 |
