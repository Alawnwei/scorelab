## 二、新发现的管线问题

### 问题 A（P1）：BTTS 系统性低估 8-12pp

**位置：** `predict.py` → `nb_bttts()` 第 114-121 行

```python
def nb_btts(lam_h, lam_a, theta, mg=6):   # mg=6 太短！
    prob = 0.0
    for i in range(1, mg+1):
        pi = nb_prob(i, lam_h, theta)
        for j in range(1, mg+1):
            pj = nb_prob(j, lam_a, theta)
            prob += pi * pj
    return prob
```

**根因：** mg=6 截断对于 λ≥2.5 的比赛丢失大量概率质量。NB(θ=8, λ=2.7) 下 P(7+)≈4-6%，截断后 P(≥1)**从 0.90 降到 0.86**，两队乘积放大了误差。

**量化影响（阿根廷 λ=2.66 vs 法国 λ=2.74）：**

| 指标 | 当前(mg=6) | 正确值(mg=15) | 误差 |
|:-----|:----------:|:-------------:|:----:|
| P(阿根廷≥1) | 0.859 | 0.900 | -4.1pp |
| P(法国≥1) | 0.850 | 0.907 | -5.7pp |
| **BTTS** | **0.730** | **0.816** | **-8.6pp** |

报告显示 BTTS 74.3%，实际应为 81-83%。对于高 λ 比赛，BTTS 被持续低估。

**修复方向：** `mg` 根据 λ 动态缩放：`mg = max(6, int(lam * 3 + 4))`

### 问题 B（P2）：P_base 与 NB 双模型未做一致性校验

**位置：** `predict.py` 第 232-240 行

**当前流程：**
```
P_base (7维评分差 × logistic) → P_final (× M)
NB probs (λ 驱动独立计算)
方向 = NB probs 阶梯判断
```

两个模型使用不同输入：
- P_base 来自 7 维评分分差（评分制）
- NB probs 来自 λ 值（xG_for 经 adjust_lambda 调整）

**问题：** 两个模型可以给出冲突信号。

**假设场景：**
```
P_base = 0.55 (7维判定主队略优)
NB probs = (42%, 25%, 33%) (λ 判定接近均衡)
→ 方向 = '主胜' (因 42 > 33 且拉开平局)
→ P_final = 0.55 × 0.9625 = 0.54 (转化后)
```

P_final=0.54 暗示 54% 主胜概率，但 NB 说只有 42%。没有一致性检查。

**修复方向：** 计算 P_final 和 NB probs 的 KL 散度或简单差值绝对值，超过阈值（如 15pp）时触发矛盾降级信号。

### 问题 C（P2）：adjust_lambda 仅考虑对手防守，忽略本方进攻 ✅ 已修复

**位置：** `predict.py` 第 127-134 行

**问题：** `own_att_str` 形参传入了但不使用，只看对手防守忽略本方进攻强度。

```python
# 旧：只看防守

### 13. adjust_lambda 修复——攻防各自独立修正（P0 Hotfix）

**问题：** `ratio = own_att_str / opp_def_str` 导致强防使 λ 升高，埃及 λ=3.61 > 阿根廷 3.33 不合理。

**根因：** 除法使 `opp_def_str < 1.0`（好防守）→ ratio > 1 → λ 升——**逻辑反了**。

**修复：** 改用加法 `own_att_effect + opp_def_effect`，各自独立修正：

```
