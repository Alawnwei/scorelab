# 修复 ELO 缓存覆盖与过期回退 — 变更说明

> 修复 `_load_elo_rankings` API 请求异常覆盖磁盘缓存，以及 `get_team_elo` 无过期缓存回退的问题。

---

## 问题

### 问题一：API 异常数据覆盖磁盘缓存

`_load_elo_rankings` 在 API 返回非空但列名不匹配的 DataFrame 时（例如 `country` → `Country`），`_ELO_CACHE` 保持为空 `{}`，但 **无条件写入磁盘**（`datafc_provider.py:1099`），覆盖了之前有效的缓存数据。

触发条件：
- `eloratings.world_ranking_data()` 返回格式变更（大小写、列名）
- 网络异常导致响应为 HTML/错误页（非空但无法解析）
- 上述情况均会写入空 `{}` → 下次读缓存时 `_data.get(code)` 返回 `None`

### 问题二：过期缓存无回退路径

`get_team_elo` 严格校验缓存 TTL（`_age < _ELO_CACHE_TTL`），过期数据直接丢弃。当 API 不可达（`DATAFIC_ALIVE = False`）且磁盘缓存过期时，返回硬编码 `{elo: 1500}`，导致：
- 西班牙（真实 ELO ~1850+）被低估约 350 分
- 所有强队胜率系统性偏低

## 修改

### `_load_elo_rankings`

| 改动 | 行号 | 说明 |
|:-----|:----:|:-----|
| 增加 `DATAFIC_AVAILABLE` 守卫 | 1092 | 无 datafc 包时不调用 API |
| 列名兼容 `country`/`Country`, `elo`/`Elo`, `rank`/`Rank` | 1099-1101 | 兼容 API 格式变化 |
| 仅非空 `elo_map` 时写入磁盘 | 1105-1112 | 空数据不覆盖有效缓存 |
| API 失败后回退过期磁盘缓存 | 1121-1124 | `stale_data` 保存逻辑在 1087 |

### `get_team_elo`

| 改动 | 行号 | 说明 |
|:-----|:----:|:-----|
| 移除 TTL 检查 | 1147-1151 | 不过期，有数据即返回 |
| 增加 `isinstance` 类型守卫 | 1147, 1150 | 防止 JSON 格式异常导致崩溃 |

核心原则：**宁可返回 7 天前的 ELO 数据，也不返回默认 1500.**

## 影响范围

- `datafc_provider.py` 中的 ELO 相关函数
- `get_team_elo` 调用者（预测引擎中的 ELO 计算步骤）
