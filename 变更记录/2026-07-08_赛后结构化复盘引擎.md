# 赛后结构化复盘引擎

## 现状问题

pnl_records 只记录 `win/loss + 盈亏`，没有结构化的预测vs实际对比。

```
旧: 阿根廷 3-0埃及 → pnl_records: win,+0.15
新: 
  ┌─ 赛前: P_final=0.50, 客胜56.7%, 大2.5=87.4%, BTTS=80.3%
  ├─ 实际: 3-1 → 方向❌ 大2.5✅ BTTS✅
  ├─ λ偏差: 预期5.5球 vs 实际4球
  ├─ 根因: 七维评分低估主队(score_diff=-0.77)
  └─ 修正: 检查阿根廷防守评分是否站不住
```

## 改动

### 1. 预测数据增强（predict.py）

每次写入 predictions_db.json 时新增全量市场数据：

```json
{
  "lam_h": 2.2187,
  "lam_a": 3.2647,
  "probs_3way": {"home": 0.279, "draw": 0.149, "away": 0.567},
  "over_25": 0.874,
  "btts": 0.803
}
```

### 2. 结构化复盘引擎（post_match_analysis.py 新增）

```
python skill/post_match_analysis.py          # 复盘最近7天未复盘记录
python skill/post_match_analysis.py --all    # 复盘全部
python skill/post_match_analysis.py --match 巴西  # 复盘指定比赛
```

复盘报告输出到 `预测数据/结构化复盘-YYYY-MM-DD.md`

**复盘内容**：

| 分析维度 | 数据来源 |
|:---------|:---------|
| 方向准确率 | DB correct 字段 / direction 推断 |
| 1X2 各选项概率偏差 | probs_3way vs 实际结果 |
| 大小球偏差 | over_25 vs actual goals |
| BTTS 偏差 | btts vs actual both scored |
| λ 偏差 | lam_h+lam_a vs 实际总进球 |
| 方向偏差 | score_diff vs 实际胜负关系 |

**根因归类**：
- `λ偏差`：预期进球偏离 >2.0球 → 检查防守评分
- `方向偏差`：预测方向与实际相反 → 七维评分可能高估
- `大小球/BTTS偏差`：概率偏离 >20pp → 淘汰赛λ修正不足
- `正常波动`：所有偏差在合理范围内

### 3. 偏差数据累积

已在 predictions_db.json 中添加 `result.reviewed` 和 `result.reviewed_at` 字段，避免重复复盘。
每次复盘后生成的结构化报告自动归档到 `预测数据/`。
