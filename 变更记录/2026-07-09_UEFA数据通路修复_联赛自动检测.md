# UEFA 数据通路修复 + 联赛自动检测

> 日期: 2026-07-09
> 涉及文件: `skill/datafc_provider.py`
> 本修复已被 [v8.5 全面校准修复](2026-07-09_v8.5_全面校准修复_9项改进.md) 覆盖

---

## 问题背景

预测 Kairat Almaty vs FK Sutjeska（欧冠资格赛）时传入了 `--league uecl`（欧协联），导致两个问题：

1. **UEFA 赛事数据通路被硬编码跳过** — `get_team_recent_matches` 中 `type == "uefa"` 直接 `return pd.DataFrame()`，欧战球队的近期数据完全无法获取
2. **传错联赛代码无兜底** — 用户手动传错 `--league` 时系统无自动修正机制，只能依赖调用者回退到 OpenLigaDB 默认值

## 修复清单

### 1. 删除 UEFA 跳过逻辑（datafc_provider.py:1275-1279）

**改前**: `uefa`/`knockout` 类型联赛直接返回空 DataFrame，数据通路在入口处即被堵塞：
```python
if cfg.get("type") in ("knockout", "uefa"):
    return pd.DataFrame()
```

**改后**: 放行到 datafc 扫描，欧战球队仍能通过本国联赛数据获取近期比赛：
```python
if cfg.get("type") in ("uefa", "knockout"):
    pass  # 放行到下面的 datafc 扫描
```

**效果**: Kairat Almaty（哈萨克超）、或其他欧战球队在 Champions League / Europa League / Conference League 联赛代码下不再被阻塞。

### 2. 联赛自动检测（datafc_provider.py:2859-2930）

新增 `detect_team_league(team_name, fallback_league)` 函数：

- 通过 `dfc_search` 搜索球队的 Sofascore 实体
- 提取 `tournament_id` 反向映射回 `LEAGUE_TOURNAMENT` 中的联赛代码
- 支持 tournament_name 模糊匹配作为二次回退
- 带内存缓存，同会话不重复请求

### 3. 自动重试（datafc_provider.py:1892-1899）

在 `estimate_team_strength` 中，当指定联赛无数据时自动触发检测 + 重试：

```python
if xg.get("matches_played", 0) == 0:
    detected = detect_team_league(team_name, fallback_league=league)
    if detected != league:
        xg = get_team_xg_summary(team_name, detected, ...)
        league = detected  # 后续逻辑用新联赛
```

**效果**: `--league uecl` + Kairat Almaty → 自动检测为 `ucl` → 用欧冠 tid=7 获取数据

## 影响范围

| 场景 | 改前 | 改后 |
|------|------|------|
| `--league uecl` + 欧冠球队 | 数据源 `web_scout`，默认 λ | 自动检测为 `ucl`，真实数据 |
| `--league uel` + 欧联球队 | 同上 | 同上 |
| `--league ucl` + 欧冠小组赛球队 | 数据通路跳过 | datafc 正常扫描 |
| 非欧战联赛 | 不受影响 | 不受影响 |
