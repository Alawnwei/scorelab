# Git 架构 + HTML 看板 + 通知方案

## 架构：Git 作为"服务器"

```
你的笔记本 → Gitee（代码+数据）
     ↓
本地跑预测 → HTML 看板 → git push
     ↓
Gitee Pages 托管 → 手机/电脑都能看
     ↓
赛后自动回填 → 更新看板 → git push
```

## 新增：HTML 看板生成

```bash
python skill/report_to_html.py
```

生成 `html/report.html`（19KB 自包含文件），包含：

| 模块 | 内容 | 技术 |
|:-----|:-----|:-----|
| KPI 卡片 | 准确率/Brier/LogLoss/PnL/待结算 | CSS Grid |
| 校准曲线 | 实际胜率 vs 预测均值柱状图 | Chart.js |
| 趋势图 | 滚动窗口准确率变化 | Chart.js |
| 待结算预测 | 当前未出结果的预测 | HTML table |
| 最近结算 | 最近30场预测vs实际 | HTML table |
| 深色/浅色 | 响应式主题 | CSS prefers-color-scheme |

## 每日操作流

```bash
# 早上：拉最新数据
git pull

# 预测
bash run_predict.sh --home "葡萄牙" --away "西班牙" --league wm26

# 生成看板并推送
python skill/report_to_html.py
git add -A
git commit -m "2026-07-08 预测更新"
git push

# 手机打开 Gitee Pages 查看
```

## Gitee 配置

### Gitee Pages
1. 仓库 → 服务 → Gitee Pages
2. 选择 `html/` 目录
3. 启用后访问：`https://你的用户名.gitee.io/足球分析/html/report.html`

### 通知方案
- **方案 A**：Gitee 仓库 Watch → 每次 push 邮件通知到手机
- **方案 C**：本地 `--watch` 常驻检测到 +EV → Windows 弹窗
