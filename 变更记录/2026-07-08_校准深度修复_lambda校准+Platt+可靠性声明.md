# 2026-07-08 校准深度修复

> 基于审计指出的"校准断裂"问题实施的 3 项修复。  
> 核心目标：将校准从 1X2 市场扩展到所有市场，并让用户可见校准可靠性。

---

## 修复 1：λ 全局校准（predict.py）

### 问题

温度缩放 T=4.08 仅影响 P_final（1X2 胜率），但大小球/BTTS/亚盘的概率来自 λ → 负二项分布，**从未被校准**。

### 修复

新增 `_load_lambda_calibration()` 函数：

```python
def _load_lambda_calibration():
    """从已结算记录计算 λ 校准因子
    因子 = 实际场均总进球 / 模型预期场均总进球
    """
```

在 `adjust_lambda()` 之后应用：

```python
_lam_calib_factor = _load_lambda_calibration()
if _lam_calib_factor is not None:
    lam_h *= _lam_calib_factor
    lam_a *= _lam_calib_factor
```

### 当前结果

| 指标 | 值 |
|:-----|:---:|
| λ 校准因子 | **1.2821** |
| 含义 | 实际进球比默认基线高 28% |
| 触发条件 | ≥10 条已结算记录 |
| 偏差 >5% 时启用 | ✅ 启用中 |

⚠️ 当前基线使用硬编码 `2.5`（五大联赛典型值）。世界杯实际场均更高。  
未来改进：使用 league-specific 基线。

## 修复 2：校准可靠性声明（预测报告输出）

### 问题

用户无法判断一个预测的 +EV 值是"经过数据验证"的还是"纯模型猜测"。

### 修复

在所有预测报告的末尾新增"校准可靠性声明"区块：

**当校准数据不足（<10条）：**
```
🔴 校准数据不足（已结算记录 < 10 条）
所有概率为原始模型输出，+EV 计算仅供参考。
```

**当系统性偏差被修正：**
```
🟡 全局 λ 校准已启用（因子=1.282）
模型存在系统性偏差(λ高估0%)，已自动修正。
```

**当校准正常：**
```
✅ 概率已校准（λ因子=1.000, T=4.08）
Brier=0.2624, 校准样本=16条。
```
