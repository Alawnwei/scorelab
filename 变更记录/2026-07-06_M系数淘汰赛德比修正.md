# M系数自动推断 — 淘汰赛德比修正

> 修复淘汰赛德比场景下 M 系数为 1.0（无修正）的问题。
> 新增：历史交锋权重 ×0.625、德比叙事溢价 +0.5球。

---

## 问题

`infer_m_factors` 仅返回轮换/大败/休息差三个因子，未区分赛事类型（淘汰赛 vs 小组赛）和比赛性质（德比 vs 普通），导致：

- 世界杯/杯赛淘汰赛德比 → M 系数 = 1.0（无任何场景修正）
- 历史交锋权重未被计入（SKILL.md 规范要求淘汰赛 ×0.625）
- 德比战叙事驱动没有被量化（复仇/纪录/荣誉 → 进球预期增加）

## 修改

### 新增函数

| 函数 | 行号 | 说明 |
|:-----|:----:|:-----|
| `is_knockout_league(league)` | 1387 | 基于 `LEAGUE_TOURNAMENT['type']` 判断淘汰赛制 |
| `is_derby_match(home, away)` | 1404 | 基于 `DERBY_PAIRS` 表判断经典德比（含20组国家队德比） |
| `calc_h2h_weight(league)` | 1451 | 小组赛 1.0 / 淘汰赛 0.625（WVR衰减后） |
| `calc_derby_goal_bonus(home, away)` | 1462 | 德比 +0.5 球预期 |

### `infer_m_factors` 新增返回值

| Key | 类型 | 取值 | 意义 |
|:----|:----:|:-----|:-----|
| `league_type` | str | `group` / `knockout` | 赛事阶段，防默认 1.0 |
| `h2h_weight` | float | `1.0` / `0.625` | 历史交锋权重因子 |
| `must_win` | str | `none` / `low` / `high` | 战意推断 |

### 调用路径

```
predict.py（编译） → infer_m_factors(team, league)
                       ├─ rotation    (已有)
                       ├─ big_loss    (已有)
                       ├─ rest_diff   (已有)
                       ├─ league_type ← is_knockout_league()
                       ├─ h2h_weight  ← calc_h2h_weight()
                       └─ must_win    ← is_knockout_league() ? high : none

预测引擎 → calc_derby_goal_bonus(home, away) → +0.5球（德比叙事）
```

## 影响范围

- `datafc_provider.py`: 新增 4 个函数 + 增强 `infer_m_factors`
- 预测引擎中 M 系数计算：淘汰赛德比不再为 1.0
- 大小球预测：德比战自动 +0.5 球预期
