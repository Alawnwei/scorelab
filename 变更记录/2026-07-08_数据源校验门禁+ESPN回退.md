# 数据源校验门禁 + ESPN 回退通路

## 1. 数据源校验门禁（predict.py）

所有预测运行前必须先校验联赛是否有数据源覆盖。

```bash
# 无数据源的联赛 → 被拦截
python predict.py --home A --away B --league afcc2
# ❌ 联赛 afcc2 无数据源覆盖
#    无数据支撑的预测将污染校准数据，已阻止运行。

# 有数据源的联赛 → 正常运行
python predict.py --home "葡萄牙" --away "西班牙" --league wm26
# ✅ [数据源] wm26 → OpenLigaDB
```

**覆盖规则**：

| 数据源 | 适用联赛类型 | 示例 |
|:-------|:------------|:------|
| datafc (Sofascore) | league, knockout | 五大联赛、北欧、日韩、中超... |
| OpenLigaDB | world_cup, uefa | 世界杯、欧冠、欧洲杯、美洲杯... |
| ESPN | 配置了 `espn` slug 的联赛 | 亚冠精英、亚足联杯 |

## 2. ESPN 回退通路

为 AFC 赛事（亚冠精英、亚足联杯）添加：
- `LEAGUE_TOURNAMENT` 新增 `espn` 字段 → 指向 ESPN API slug
- `_espn_check_match()` → 赛程校验 ESPN 回退
- `is_match_scheduled()` → OpenLigaDB 无数据时自动查 ESPN
