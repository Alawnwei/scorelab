## 一、校准数据过滤修复（`update_calibration.py`）

### 1. 数据污染问题

**问题：** `load_valid()` 的过滤条件 `p["p_final"] == 0.500 and not p.get("direction","") and not p.get("is_bet")` 只排除了极少数的"完全空白"占位记录。遗留迁移的 **16 条 `p_final=0.500, p_final_source=inferred_neutral`** 记录全部混入校准集，导致 Platt Scaling 搜索到 A=0.00（完全忽略模型输出，返回常数概率 ≈84%）。

校准集中 16/31 = 52% 的数据是无效占位。

**修复：** 排除条件改为 `if source == "inferred_neutral" or p["p_final"] == 0.500: continue`

**影响：** 校准有效数据从 31 条 → 15 条（剔除 16 条占位），原始 Brier 从 0.4591 → 0.2991。

### 2. Temperature Scaling 自动搜索

**问题：** TEMP_T=1.60 是 2026-07-06 的经验值，从未随数据更新。

**修复：** 新增 `find_best_temperature()` 函数，网格搜索 T∈[0.20, 4.00]，使用 Brier Score 最优选择。

**结果：** 最优 T=4.08，Brier=0.2606（当前 T=1.60 时 Brier=0.2798，改善 +6.9%）

```python
# 新增函数
def find_best_temperature(valid):
    """网格搜索 0.20~4.00 找最优温度 T"""
    for t in [i*0.05 for i in range(4, 81)]:
        brier = calc_brier(valid, lambda pf, tt=t: temp_transform(pf, tt))
```

### 3. 校准报告模式（`--report`）

**新增：** 输出详细校准诊断：
- 原始概率分段校准表（`[0.00-0.30)`, `[0.30-0.40)`, ... 每段场次/正确数/实际胜率/偏差）
- 经 T=4.08 缩放后的可靠性曲线（ASCII 条形图）
- 每条校准数据明细（主队/客队/P_final/方向/结果/校正后概率）

### 4. 温度更新到 predict.py

`update_predict_temp()` 写入 `predict.py` 的 TEMP_T 参数：

## 二、结果同步脚本加固（`sync_results.py` v1.0 → v1.1）

### 问题：3 类崩溃

| 触发条件 | 根因 | 影响 |
|:---------|:-----|:-----|
| 打印 emoji（⏭️） | GBK 编码不支持 Unicode | 整个脚本退出，0 条更新 |
| SportsDB 网络超时 | 无 try/except 覆盖 | 脚本中断 |
| JSON 文件损坏 | 裸加载无容错 | 脚本中断 |

### 修复

1. **`safe_print()`** —— 替代所有裸 `print()`，三级回退：UTF-8 → GBK → ASCII
2. **`load_json()`** —— 容错 JSON 加载，文件不存在/损坏返回默认值
3. **全流程 try/except** —— main() 外层包裹，异常不崩溃，输出错误信息
4. **新增 ESPN 回退数据源** —— 自动从 `site.api.espn.com` 获取世界杯赛果，补全 auto_results.json
5. **修正 `determine_result()`** —— 增加"不败"类双向预测、大小球、混合方向的分歧判定
