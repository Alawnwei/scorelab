## P2.1 真实资金曲线追踪

### 问题
系统使用 `stake=1.0`（抽象单位）和 `stake=100`（虚构金额）两种注码语义，无真实资金追踪。

### 修复

**pnl_tracker.py** 新增：

1. **`RECORD_TEMPLATE["bankroll"]`** — 存储本金配置：`initial/current/peak/lowest/max_drawdown`
2. **`_compute_bankroll_curve(records)`** — 按结算顺序计算资金曲线节点序列
3. **`--bankroll N`** — 初始化/重置本金为 N 元
4. **`--capital`** — 显示资金曲线（ASCII 图表 + 最大回撤）

### 命令行示例

```bash
python skill/pnl_tracker.py --bankroll 10000    # 设置本金1万元
python skill/pnl_tracker.py --capital            # 查看资金曲线
```

## P2.2 自动化赛后复盘管线

### 问题
每次结果同步后需人工编写复盘报告，无自动归因。

### 修复

新增 **`skill/auto_review.py`**：

| 功能 | 说明 |
|:-----|:------|
| `--days N` | 复盘最近 N 天（默认7天） |
| `--match X` | 复盘指定比赛的错误 |
| `--all` | 全部已结算记录 |
| 输出 | Markdown 复盘报告 → `预测数据/复盘-日期-自动.md` |

### 自动归因逻辑

每条错误预测执行 `analyze_error()`：
- **L0**: P_final<0.35 的正常小概率事件
- **L1**: 七维评分数据质量、淘汰赛λ修正、进球数偏差
- **L2**: ⭐⭐⭐ 高信心出错 → 场景修正过度自信

### 输出示例

```markdown
## ❌ 错误归因
### 🟡 巴西 vs 挪威
| 预测方向 | 平局/均衡 | P_final | 0.536 |
| 实际比分 | 1-2 | 严重级别 | L1 |
**根因分析:** 七维评分极端值或数据质量不足(#026)
**改进建议:** 足球固有随机性，持续追踪

## 🔍 模式发现
| 错误模式 | 出现次数 |
| 七维评分极端值或数据质量不足(#026) | 10 |
| 高进球比赛(≥4球)，防守评分可能低估 | 2 |
```

## P2.5 偏差统计审计

### 问题
25+ 条偏差（大部分 C 级）无统计显著性检验，无法区分"真正有预测价值的模式"和"过拟合噪音"。

### 修复

新增 **`skill/deviation_audit.py`**：

```
python skill/deviation_audit.py
```

**审计 4 个维度：**

| 维度 | 方法 | 输出 |
|:-----|:-----|:------|
| P1: 概率校准 | 二项检验 per-P_final 区间 | 显著/参考/不显著 |
| P2: 联赛迁移 | 各联赛命中率 vs 全局基线 | ⚠️ 偏差标记 |
| P3: ⭐评级校准 | 实际胜率 vs 隐含概率 + 二项检验 | 校准因子 |
| P4: 偏差存活 | 活跃偏差追踪 | 已验证/未验证 |

### 关键发现

| 统计量 | 值 |
|:-------|:---:|
| 总体命中率 | 29/39 = 74.4% |
| 概率校准显著偏差 | [0.00-0.30) 区间 p=0.022（低概率低估） |
| 评级校准显著偏差 | 无评级标记→实际73.0% vs 默认55%，p=0.019 |
| 偏差存活 | 31/31 条从未触发 |

### 结论
当前 39 场样本下，统计检验为参考性，非结论性。  
建议积累至 100+ 场后重跑。
