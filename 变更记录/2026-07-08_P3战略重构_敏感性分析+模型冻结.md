## P3.1 七维评分权重敏感性分析

### 新增工具

**`skill/sensitivity_analysis.py`** — 分析各维度对预测结果的实际贡献度。

### 分析维度

| 分析 | 方法 | 产出 |
|:-----|:-----|:------|
| 权重体系 | 当前 7 维权重分布 | 核心/辅助/边缘 分级 |
| 区分度 | MD报告提取七维评分 → 各维度标准差 | 维度区分度排行 |
| P_final 区分度 | 上半区 vs 下半区正确率对比 | 整体七维体系有效性 |
| 维度精简模拟 | 用不同权重组合计算平均|得分差| | 推荐精简方案 |

### 关键发现

```
P_final 中位线: 下半区正确率=89.5%, 上半区=60.0%
区分度: ⚠️ 差
⚠️ P_final 几乎无区分度 → 七维评分整体失效风险
```

即：模型低置信度时（下半区）反而更准——说明**七维评分系统整体区分度不足**。

### 推荐精简方案

从 7 维 → **5 维**（移除 X因素 和 体能/状态）：

| 维度 | 当前 | 推荐 | 原因 |
|:-----|:---:|:----:|:------|
| 进攻能力 | 20% | **25%** | 进球是最终裁判 |
| 防守稳固性 | 20% | **25%** | 零封能力最关键 |
| 中场控制力 | 15% | **20%** | 控球+转换 |
| 大赛经验 | 15% | **18%** | 淘汰赛已验证 |
| 战术适配度 | 15% | **12%** | 风格克制难量化 |
| 体能/状态 | 10% | **移除** | 5场指数加权=近况重复 |
| X因素 | 5% | **移除** | 纯主观，从不打>6分 |

> ⚠️ 当前 39 场样本不足以做统计显著的回测。
> 建议积累至 200+ 场后运行 `--simulate` 正式切换。

## P3.3 自动模型冻结触发

### 新增强工具

**`skill/freeze_monitor.py`** — 监控预测准确率，自动触发模型冻结。

### 触发条件

| 参数 | 默认值 | 说明 |
|:-----|:------:|:------|
| `accuracy_baseline` | 65% | 低于此值触发警告 |
| `consecutive_below_baseline` | 3 场 | 连续低于基线 N 场触发冻结 |
| `min_samples` | 10 场 | 最少样本数才触发检查 |

### 命令

```bash
python skill/freeze_monitor.py              # 检查健康状态
python skill/freeze_monitor.py --freeze     # 手动冻结
python skill/freeze_monitor.py --unfreeze   # 解冻
python skill/freeze_monitor.py --baseline 0.70  # 设基线为70%
```

### 建议集成

将 `freeze_monitor.py` 加入预测管线：
- 每次 `predict.py` 运行后执行一次 `freeze_monitor.py`
- 若返回 frozen=True → 预测结果标注"⚠️ 模型已冻结"并停止自动推荐
