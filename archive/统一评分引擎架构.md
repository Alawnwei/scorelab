# 统一评分引擎（Unified Grading Engine）v7.3

## 核心问题
原系统10模块分析（定性/抓矛盾）与七维评分模型（定量/算实力）是两条平行线，没有定义数学交汇点。当七维模型说"A队胜率65%"而模块分析说"A队无优势"时，分析师只能凭感觉二选一。

## 修复方案：三级递进架构

```
L1: 七维基础实力层 → P_base (0~100%)
L2: 场景修正系数层 → M系数 (0.5~1.8)，连乘各场景因子
L3: 偏差截断与矛盾熔断 → P0级检查，#008强制移除
```

## 核心公式（赔率比空间映射法）
```
OddsRatio_base = P_base / (1 - P_base)
OddsRatio_adj  = OddsRatio_base × M
P_final        = OddsRatio_adj / (1 + OddsRatio_adj)
```

## 关键修正（审阅中发现）
1. **公式③修复**：原文案用十进制赔率（Odds=1/P）导致P_final=P_base×M可>100%，改为赔率比(Odds Ratio)形式
2. **历史交锋权重因子加入M**：偏差#001在M系数中增加小组赛1.0/淘汰赛0.85
3. **防重复规则**：L1七维与L2 M系数重叠维度（战术适配度、轮换状态）使用"去重后基线"
4. **最高位约束**：单场不得同时取3个及以上>1.10的高位因子
5. **M截断范围**：0.5~1.8

**Why:** 消除双轨并行的歧义空间，将⭐评级从自由心证改为硬映射。

**How to apply:** 每次预测中，先算L1七维评分得P_base，再算L2各场景因子连乘得M，经Odds Ratio映射得P_final，最后查评级映射表出⭐。
