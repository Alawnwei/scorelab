## 🔧 修复1：主场连胜概率回归因子（v6.0新增）

### 问题描述

| 场次 | 主场记录 | 预测 | 实际 | 偏差 |
|:----|:---------|:----:|:----:|:----:|
| AC奥卢 vs 拉赫蒂 | 主场5连胜🔥 | 主胜 ⭐⭐⭐ | **1-1平局** | ❌ |
| ~~赫尔辛基 vs 库奥皮奥~~ | ~~传统主场豪门~~ | ~~库奥皮奥不败(含平)~~ | ~~0-4惨败~~ | ~~已移至偶发事件~~ |
| 坦佩雷山猫 vs 塞伊奈 | 主场3连胜 | 山猫不败 ✅ | 2-2平局 | ✅勉强对 |

**共同模式：** 主场连胜越长，模型越相信"主场龙属性"，忽略了**连胜必然伴随概率回归**。

### v6.0规则

```diff
+ 主场连胜衰减因子（在胜平负评估中应用）

+ 连胜场次 → 信心折扣表：
+   | 主场连胜场次 | 主胜信心折扣 | 说明 |
+   |:-----------:|:-----------:|:------|
+   | 0-2场(正常) | 不折扣 | 正常主场优势 |
+   | 3-4场(热门) | -0.5档 | 接近回调临界点 |
+   | 5-7场(高危) | -1档 | 强力回归信号！历史统计6场+连胜终结概率>60% |
+   | 8场+(极端) | -1.5档或反转 | 随时可能断 |

+ 应用方式：
+   当主队连胜≥4场 → "主胜"方向不可给⭐⭐⭐
+   当主队连胜≥5场 → "主胜"最高⭐⭐，且必须标注"连胜概率回归警告"
+   当主队连胜≥7场 → "主胜"方向必须以"谨慎"为前提

+ 例外（不折扣或仅-0.5档）：
+   ① 双方实力差距悬殊（如榜首vs垫底，积分差≥15分）
+   ② 客队也有严重伤病/停赛
+   ③ 该联赛整体主胜率显著高于正常水平（如>48%）
```

### 应用验证

```
AC奥卢vs拉赫蒂（v5.0 → v6.0）：
  v5.0: "主场5连胜🔥" → 主胜⭐⭐⭐
         ↓ 忽略了5连胜已处于高概率回归点
  
  v6.0: 主队连胜=5场 → 进入"高危"(5-7场)区间
         → 主胜从⭐⭐⭐降为⭐⭐
         → 要求强制标注"连胜可能终结"警告
         → 平局方向从备选提升为与主胜并列
```

## 🔧 修复2：大败后连续低迷规则（v6.0新增）

### 问题描述

| 场次 | 上轮结果 | 预测 | 实际 | 偏差 |
|:----|:---------|:----:|:----:|:----:|
| AC奥卢 vs 拉赫蒂 | 上轮**1-5惨败**VPS | "反弹赢球" | **1-1继续低迷** | ❌ |
| 赫尔辛基 vs 库奥皮奥 | 上轮1-1战平 | 无明显异常 | 0-4惨败 | — |

**v5.0的判断逻辑：**
> "主场全胜vs客场1胜的差距不可忽视→上轮惨败更像偶发事件"

**实际：** 惨败不是偶发，而是**状态低迷的表现**。大败后的球队通常不会立即反弹。

### 统计参考

| 大败幅度 | 下一场赢球概率 | 下一场再次大败概率 |
|:--------:|:-------------:|:-----------------:|
| 输1球 | 45-50%（正常） | 5-8% |
| 输2球 | 35-40%（↓） | 10-15% |
| **输3球+（含3球）** | **25-35%（显著↓）** | **15-25%（显著↑）** |
| 输5球+（惨案） | 20-30%（极低） | 20-30% |

> 来源：欧洲主流联赛统计，大败（输3球+）后的下一场赢球概率显著低于正常水平。
> 特别是当大败发生在**主场**，士气打击更大。

### v6.0规则

```diff
+ 大败后连续低迷规则：

+ 定义"大败"：输球分差≥3球（如0-3、1-4、1-5）
+
+ 大败后的下一场评估调整：
+   ① 该队"胜"的方向信心降1档（⭐⭐⭐→⭐⭐、⭐⭐→⭐）
+   ② 不支持将大败简单归因为"偶发事件"
+   ③ 强制检查该队防守数据（大败通常暴露防守漏洞）
+      - 如果防守数据也差（近3场场均失球≥2）→ 该队"胜"不可给⭐⭐⭐
+      - 如果防守数据正常（近3场失球≤1）→ 大败可能真的是偶发，折扣可缩小至-0.5档
+
+ 例外情况（不折扣）：
+   ① 大败对手是绝对强队（如榜首/卫冕冠军）→ 可以理解
+   ② 大败中有红牌/点球等偶然因素 → 不是真实实力体现
+   ③ 大败后核心球员复出 → 实力回升
```

### 应用验证

```
AC奥卢vs拉赫蒂（v5.0 → v6.0）：
  v5.0: "上轮惨败更像偶发事件" → 主胜⭐⭐⭐
         ↓
  v6.0: 上轮1-5惨败 = 输4球 → 进入"惨案"区间
         防守检查：AC奥卢近3场失球 = 1-5-1 = 场均2.3球 ≥ 2球
         → 防守确实有问题
         → 主胜从⭐⭐⭐降为⭐⭐
         → 需要标注"大败后不必然反弹"
```

## 🔧 修复3：净胜球战意/冠军争夺战加成（v6.0新增）

### 问题描述

| 场次 | 形势 | 预测 | 实际 | 偏差 |
|:----|:-----|:----:|:----:|:----:|
| 库奥皮奥 vs 赫尔辛基 | 库奥皮奥与榜首同分(27分) | 库奥皮奥不败(平/负) | **4-0狂胜** | ❌方向对但幅度严重低估 |

**v5.0的判断：** "库奥皮奥8场不败+6次H2H对赫尔辛基不败"→不败方向正确
**但缺失的维度：** 库奥皮奥与榜首同分，但**净胜球可能处于劣势**，所以不仅需要赢，还需要大胜。

### v6.0规则

```diff
+ 净胜球战意评估（在联赛后期/争冠关键期使用）：

+ 触发条件（同时满足）：
+   ① 联赛已进行过50%赛程（如芬超14轮中的13轮→已过50%✅）
+   ② 球队在争冠区（前3）或保级区（后3）
+   ③ 球队与同分/同排名对手的净胜球差距≤3球
+
+ 评估调整：
+   - 满足以上条件的球队→"大胜"方向的证据权重增加
+   - 竞彩数据中的"让球胜"(赢2球+)应给予更高关注
+   - 典型信号：该队近期比赛进球数增加（说明在刻意刷净胜球）
+
+ 应用方式：
+   ① 如判定球队在"追净胜球" → "让球胜"方向信心上调0.5档
+   ② 如判定球队在"守净胜球"（领先优势大）→ "胜"方向维持，"让球胜"信心下调0.5档
+   ③ 净胜球战意不改变"胜平负"方向，只改变"赢球幅度"预期
+
+ 联赛阶段判定：
+   - 赛程<30% → 净胜球战意低（赛季初期，不在意净胜球）
+   - 赛程30-60% → 净胜球战意中等（开始关注排名细节）
+   - 赛程>60% → 净胜球战意高（争冠/保级关键期）
```

### 应用验证

```
库奥皮奥vs赫尔辛基（v5.0 → v6.0）：
  v5.0: 库奥皮奥不败(平/负) → 命中✅，但没想到会4-0
         ↓
  v6.0: 触发净胜球检查：
         ① 芬超14轮中的13轮 > 50% ✅
         ② 库奥皮奥排名第2(争冠区) ✅
         ③ 与榜首同分27分 → 净胜球可能决定排名 ✅
         → 判定：库奥皮奥有"刷净胜球"动机
         → "让球(-1)客胜"信心从⭐⭐上调为⭐⭐⭐
         → 比分会从2-1/1-1上调至3-0/4-0级别
```

## 🔧 修复4：得势不得分修正因子（v6.0新增）

### 问题描述

| 场次 | 角球 | 射门(预估) | 控球(预估) | 比分 | 偏差 |
|:----|:----:|:----------:|:----------:|:----:|:----:|
| AC奥卢 vs 拉赫蒂 | **8:1碾压** | 场面统治 | 场面统治 | **1-1** | ❌预测2-1 |
| 赫尔辛基 vs 库奥皮奥 | 8:9(接近) | 不清楚 | 不清楚 | 0-4 | ❌不同情况 |

**AC奥卢的8:1角球是最极端的"得势不得分"案例。**
角球8:1说明奥卢全场压制，但只进1球→**进攻效率极低**。

### 角球与进球的关系模式

从芬超+世界杯数据总结，角球比与进球的关系有三种典型模式：

| 模式 | 角球比特征 | 进球特征 | 典型场景 |
|:----|:----------|:---------|:---------|
| **模式A：统治=高效** | 角球比≥2:1且总角球≥10 | 统治方进2-3球 | 实力悬殊+进攻流畅 |
| **模式B：统治=低效** | 角球比≥3:1但统治方≤1球 | 统治方进球少 | 防守方密集+统治方效率差 |
| **模式C：均衡对攻** | 角球比接近(如8:9) | 进球分散 | 实力接近+互有攻守 |

### v6.0规则

```diff
+ 得势不得分修正因子（在大小球和比分预测中使用）：

+ 核心逻辑：角球优势 ≠ 进球优势
+   统治方角球≥对方2倍但只进≤1球 = "得势不得分"信号
+
+ 触发条件（需要赛前数据支持）：
+   ① 统治方近期比赛中，角球≥6但进球≤1的场次≥2次
+   ② 或统治方近期射门转化率低于联赛平均（如<10%）
+   ③ 或对方防守组织数据优异（如近期被射门多但丢球少）
+
+ 调整方式：
+   触发后→大球预期下调0.5-1球
+   触发后→"大2.5球"方向不可给⭐⭐⭐
+   触发后→统治方"让球胜"方向降1档
+
+ 赛后复盘标记：
+   角球比≥3:1但比赛总进球≤2 → 标记为"得势不得分"案例
+   积累足够案例后，建立角球-进球转换效率基准
```

### 实际应用限制

这个因子在赛前**需要统治方的射门转化率数据**，目前模型尚未系统采集该数据。
v6.0阶段先做**赛后标记累积**，待数据采集完善后再转为赛前预测因子。

## 🔧 修复5：[芬超] 中游联赛保守化规则（v6.0新增）

### 问题描述

芬超（以及类似的中游联赛）与世界杯有本质不同：

| 维度 | 世界杯 | 芬超/中游联赛 |
|:----|:------|:-------------|
| 数据可靠性 | 高（国家队数据稳定） | **低（样本小、波动大）** |
| 排名稳定性 | 相对稳定 | **排名每周大变** |
| 主场优势 | 中立场地 | **主场优势显著但波动** |
| 信息透明度 | 高（全球媒体覆盖） | **低（本地联赛信息不对称）** |
| 赔率精算 | 精确（全球投注） | **较粗略（投注量小）** |

### v6.0规则

```diff
+ 中游联赛保守化规则：

+ 适用联赛特征（满足任意2条）：
+   ① 联赛总轮次≤30（样本小）
+   ② 球队间实力差距小（无绝对统治力球队）
+   ③ 主场胜率异常高（>50%）或异常低（<35%）
+   ④ 信息获取难度高（非英语/非主流联赛）
+
+ 调整方式（在适用联赛中应用）：
+   ① 任何"胜"方向默认-0.5档信心（⭐⭐⭐→⭐⭐、⭐⭐→⭐）
+   ② 让球盘口不做"穿盘"推荐（只做"赢盘"方向的判断）
+   ③ 大小球倾向保守（更倾向小2.5，除非有极强证据）
+   ④ 比分预测区间扩大（±1球→±1.5球）
+   ⑤ 避免使用"最稳方向"标签（⭐⭐⭐最高只能给⭐⭐）
+
+ 芬超当前判定：触发保守化 ✅
+   - 总轮次22轮（<30）✅
+   - 无绝对统治力球队（榜首27分但库奥皮奥同分）✅
+   - 本土联赛，信息获取有限 ✅
```

## 🔧 修复6：赛后偏差分类规则 — 偶发事件 vs 系统性偏差（v6.0新增）

### 问题描述

赫尔辛基vs库奥皮奥在复盘时误标为🔴大偏差，但实际是因为**开场12分钟红牌→78分钟10打11**，属于不可预测事件。

如果所有偏差都"一刀切"归为模型问题，会导致两个后果：
1. **过度修正** — 为偶发事件修改模型，反而降低预测质量
2. **真实问题被稀释** — 系统性偏差的真实信号被偶发事件掩盖

### 分类标准

赛后分析时必须先判断偏差性质，再决定是否触发模型修正：

| 分类 | 定义 | 案例 | 是否触发修正 |
|:----|:-----|:-----|:-----------:|
| **🔴 偶发事件** | 不可预测的随机事件，与模型逻辑无关 | 12分钟红牌、点球罚丢、门将低级失误、严重伤病开场退赛 | ❌ **不修正模型**，仅记录 |
| **🟡 边界案例** | 偶发+系统混合，有运气成分但也有模型盲区 | 争议点球、VAR改判、极端天气（暴雨/大风） | ⚠️ 评估后决定，通常不修正 |
| **🟠 系统性偏差** | 模型逻辑缺陷、数据解读错误、因子缺失 | 主场连胜高估、大败反弹误判、零封过度自信 | ✅ **必须修正模型** |

### 偶发事件判定细则

满足以下**任意1条**即判定为🔴偶发事件，偏差不计入模型问题：

```
① 开场30分钟内出现红牌（含直红+两黄变一红）
   理由：30分钟前红牌=至少60分钟人数优势，比赛失去公平性基础

② 开场15分钟内出现点球（改变比赛走势）
   理由：过早点球扭曲了预期的比赛节奏和战术部署

③ 赛前2小时内核心球员突遭伤病退赛（需有官方确认）
   理由：赛前已完成的预测无法反映这个变化

④ 门将重大失误直接导致失球（需赛后公认）
   理由：门将失误是违约事件，与战术/实力无关

⑤ 极端天气导致比赛性质改变（暴雨/暴雪/大风预警）
   理由：天气改变比赛形态（高球变多、技术下降）
```

### 复盘流程

```
赛后对比实际vs预测
        │
        ▼
  偏差是否存在？
        │
   ┌────┴────┐
   │ 否      │ 是
   │         ▼
   │    偏差原因分析
   │         │
   │    ┌────┴────┐
   │    │ 满足任意 │ 偶发事件判定条件
   │    │ 1条?     │
   │    └────┬────┘
   │    ┌────┴────┐
   │    │ 是      │ 否
   │    ▼         ▼
   │ 标记为    标记为
   │ 🔴偶发    🟠系统性偏差
   │ ❌不修正   ✅触发vN+1修正
   │             │
   │         ┌────┴────┐
   │         │ 写入    │
   │         │ vN+1方案│
   │         └─────────┘
   │
   └────→ 汇总到统计表
```

### 应用验证


## ⚠️ v6.0 应用前置条件
